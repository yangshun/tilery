# Roadmap

This roadmap tracks remaining gaps and completed parity work between `tilery`
and comparable panel systems.

## Compared Libraries

| Library                  | Why it matters                                                                                                            |
| ------------------------ | ------------------------------------------------------------------------------------------------------------------------- |
| `react-tiling`           | Original app-local target for tabs, panel modes, and app workflow parity.                                                 |
| `react-resizable-panels` | Best resize API benchmark: units, constraints, collapse, callbacks, and accessible separators.                            |
| Dockview                 | Closest IDE-layout benchmark: dock groups, floating groups, popout windows, serialization, and multi-framework core.      |
| FlexLayout               | Mature React docking benchmark: tabsets, border tabsets, popouts, submodels, overflow, and rich layout model actions.     |
| rc-dock                  | React dock-layout benchmark with controlled/uncontrolled layouts, floatbox/dockbox model, save/load, and tab update APIs. |
| GoldenLayout             | Established docking benchmark for popup windows, load/save layouts, focus, theming, and framework integration.            |
| react-mosaic             | Tree-layout benchmark for controlled/uncontrolled tiling, drag-rearrange, toolbar controls, and migration helpers.        |
| Allotment                | Splitter UX benchmark: min/max/preferred sizes, snapping, reset, visibility, and sash states.                             |
| React-Grid-Layout        | Dashboard-grid benchmark: responsive breakpoints, item constraints, static widgets, packing, and serialization.           |
| Gridstack                | Dashboard-grid benchmark: nested grids, responsive layout, drag/resize toggles, and framework-agnostic usage.             |

## Feature Gaps

| Priority | Feature                                 | Covered by                                                             | Tilery status | Remaining work                                                                                                                                                                                                    |
| -------- | --------------------------------------- | ---------------------------------------------------------------------- | ------------- | ----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| P0       | Collapsed panel modes                   | `react-tiling`, `react-resizable-panels`, Allotment, Dockview Paneview | Gap           | Add `collapsed`, `collapsedTitle`, `collapsible`, `collapsedSize`, drag-to-collapse rules, pre-collapse restore size, and panel object methods for `collapse()`, `expand()`, `isCollapsed()`, and `resize(size)`. |
| P1       | Flexible size units                     | `react-resizable-panels`, Allotment                                    | Partial       | Evolve `size` beyond percentage numbers to support pixels, percentages, font-relative units, and viewport-relative units without another rename.                                                                  |
| P1       | Container resize policies               | `react-resizable-panels`, Allotment, React-Grid-Layout                 | Partial       | Default proportional container resize behavior is implemented. Add an explicit pixel-preserving policy, resize priority controls, and previous-size restoration for advanced layouts.                             |
| P1       | Edge and border groups                  | Dockview, FlexLayout                                                   | Gap           | Add pinned edge/border tabsets or sidebars that can hold tabs independently from the main tiled grid.                                                                                                             |
| P1       | Link tabs and tab metadata              | `react-tiling`, FlexLayout, rc-dock                                    | Partial       | `renderTabTrigger` supports link/router tab triggers while keeping `href` consumer-owned in tab `data`; defer `allowOverflow` until its intended behavior is clear.                                                |
| P2       | Controlled layout mode                  | rc-dock, react-mosaic, React-Grid-Layout                               | Partial       | Decide whether `tilery` should expose a fully controlled `layout` prop in addition to `initialLayout`, or keep mutation through controller methods and callbacks only.                                            |
| P2       | External drag sources                   | Dockview, React-Grid-Layout, Gridstack                                 | Gap           | Add a documented path for dragging tabs/panels from outside Tilery into a target panel, split zone, or tab row.                                                                                                   |
| P2       | Nested Tilery instances and sub-layouts | FlexLayout, Dockview, react-mosaic                                     | Partial       | Document supported nested usage, event isolation, drag boundary behavior, and serialization strategy for nested layouts.                                                                                          |

## Done

| Feature                              | Covered by                                                                                                        | Completed work                                                                                                                                                                                                                                                                                            |
| ------------------------------------ | ----------------------------------------------------------------------------------------------------------------- | --------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| Accessible resize handles            | `react-resizable-panels`                                                                                          | Divider handles have keyboard resizing, separator ARIA state/value attributes, focus styling, and 24px targets. T-junction shortcuts stay pointer-only; the underlying dividers provide the standards-aligned keyboard path.                                                                              |
| Locking behavior controls            | `react-resizable-panels`, Allotment, rc-dock, Dockview, FlexLayout, React-Grid-Layout, Gridstack                  | Global resize disable, layout-item `resizable`/`draggable`/`droppable` locks, `locked` shorthand, tab-level `closeable`/`draggable` locks, runtime tab behavior APIs, and snapshot round-tripping are implemented.                                                                                        |
| Resize handle states and hit targets | `react-resizable-panels`, Allotment                                                                               | Resize handles expose `data-resize-active`, `data-resize-at-min`, `data-resize-at-max`, and `data-resize-disabled`; dividers expose `data-orientation`; `resizeHandleHitSize` customizes pointer hit targets. Hover and focus styling intentionally use CSS pseudo-classes.                               |
| Unit-aware resize constraints        | `react-resizable-panels`, Allotment, React-Grid-Layout, Gridstack, Dockview                                       | Per-panel `minSize` and `maxSize` constraints, plus the default `minSize` prop, accept percentage numbers, percentage strings, and pixel strings. Pixel constraints are resolved against the measured resize axis. Impossible constraints warn once in development while runtime resize clamps or no-ops. |
| Default container resize behavior    | `react-resizable-panels`, Allotment, React-Grid-Layout                                                            | Container resize preserves existing split proportions by default and normalizes split sizes only when measured min/max constraints would otherwise be violated. Impossible constraints resolve deterministically and keep development warnings.                                                           |
| Double-click divider reset           | `react-resizable-panels`, Allotment                                                                               | Docked layout items support `defaultSize` reset targets. Double-clicking a divider restores adjacent panels to their default ratio, snapshots preserve the targets, and reset uses the same min/max constraint clamping as drag resizing.                                                                 |
| Resize lifecycle callbacks           | `react-resizable-panels`, Allotment, React-Grid-Layout, Gridstack                                                 | `onResize` reports high-frequency pointer and keyboard resize changes. `onResizeEnd` reports committed resize changes with source metadata, affected panel size deltas, and measured pixel sizes when available.                                                                                          |
| Structural lifecycle callbacks       | `react-tiling`, FlexLayout, Dockview, rc-dock                                                                     | `onActiveTabChange`, `onTabsMove`, `onPanelsOpen`, `onPanelSplit`, `onTabsOpen`, `onTabsClose`, and `onPanelsClose` report active-tab, move, split, open, and close transitions with previous/next state and compact snapshots.                                                                           |
| Tab overflow and tab-row scrolling   | `react-tiling`, FlexLayout                                                                                        | Tab rows keep natural-width tabs, scroll horizontally with wheel input, keep the active tab visible, expose `data-overflowing`, and show an accessible hidden-tab overflow menu that activates and reveals selected tabs.                                                                                 |
| Layout persistence API               | Dockview, FlexLayout, rc-dock, GoldenLayout, `react-resizable-panels`, react-mosaic, React-Grid-Layout, Gridstack | `getLayout()` returns a serializable `TileryLayoutSnapshot`, `setLayout(layout)` restores one, snapshots can be passed to `initialLayout`, and the persistence example now stores snapshots directly.                                                                                                     |
| Floating detached panels             | Dockview, FlexLayout, rc-dock, GoldenLayout                                                                       | Same-document floating panels are supported through `type: "root"` layouts, floating panel snapshots, `floatPanel()`, `dockPanel()`, `focusPanel()`, `setFloatingPanelBounds()`, draggable and resizable floating chrome, z-ordering, docking back into tiled layouts, and a floating panels example.     |
| Native popout windows                | Dockview, FlexLayout, rc-dock, GoldenLayout                                                                       | Floating panels can be opened in same-origin browser windows with `popoutPanel()`, `returnPanelToFloating()`, popout snapshot metadata, copied stylesheet/link tags, owner-document-aware rendering, popup-blocker fallback, and a native popout example.                                                 |
| High-level tab workflow APIs         | `react-tiling`, FlexLayout, rc-dock, Dockview                                                                     | `openOrActivateTab(tab, target)` activates an existing stable-id tab or inserts it once near a related tab. `changeTabId(oldTabId, newTabId)` renames temporary ids while preserving tab state, panel order, active-tab references, and snapshots.                                                        |
